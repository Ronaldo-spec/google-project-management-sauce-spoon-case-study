# Google Project Management: Sauce & Spoon Case Study

A portfolio-ready project management case study based on the Sauce & Spoon menu tablet rollout scenario from Google Project Management coursework.

This repository documents project management artifacts such as project charter preparation, stakeholder analysis, project planning, quality standards, evaluation, retrospective documentation, stakeholder communication, and project closeout reporting.

---

## Project Overview

Sauce & Spoon is a small restaurant chain that wants to support annual growth and expansion by piloting menu tablets at selected restaurant locations. The tablet pilot is expected to help restaurants serve more guests in less time, provide useful operational data, improve service experience, and support business decision-making.

---

## Business Goals

- Reduce table turnover time.
- Reduce guest wait time.
- Improve order accuracy.
- Reduce negative customer reviews.
- Train staff on the new tablet workflow.
- Provide operational data for future rollout decisions.

---

## Project Management Artifacts

| Artifact | Purpose |
|---|---|
| Project Charter | Align project summary, goals, deliverables, scope, benefits, costs, and stakeholder concerns. |
| Stakeholder Analysis | Assess stakeholder power, interest, influence, and communication needs. |
| Coalition Email | Build stakeholder support for scope control and project alignment. |
| Project Plan | Identify tasks, milestones, estimates, dependencies, and confidence levels. |
| Quality Standards | Define measurable service and tablet performance expectations. |
| Evaluation Plan | Create evaluation questions and indicators for pilot success. |
| Retrospective Document | Identify what worked, what needs improvement, and key lessons learned. |
| Closeout Report | Summarize project goals, approach, results, and key achievements. |

---

## Key Planning Outputs

### Project Tasks

- Coordinate branding and customization of the tablet menu interface.
- Gather and evaluate stakeholder requirements.
- Research and select a suitable tablet vendor.
- Create a statement of work and vendor agreement.
- Install and test Wi-Fi readiness.
- Create tablet menu training material.
- Prepare troubleshooting and technical support plans.
- Conduct tablet system testing.
- Create a staff change management plan.
- Prepare standard operating procedures for tablet cleaning and maintenance.

### Quality Standards

| Quality Area | Target |
|---|---|
| Average ticket time - appetizer | 8 minutes |
| Average ticket time - entree | 12-15 minutes |
| Payment time | 1 minute or less |
| Order accuracy | 98% |
| Weekly technical issue reports | Less than 5% of customers |
| Lobby wait time | Less than 10 minutes |
